springMvc+mybatis批量查询

项目环境基于springMvc+mybatis,数据库使用的是MySql。

前端页面

页面效果如下，点击新增则增加一个相同的输入框，提交后显示多条订单的查询结果。

HTML主要代码：

<form id="search_form" action="" method="get">
    <input type="text" name="Number" class="Number">
</form>

如果不使用ajax方式提交，则直接使用按钮提交即可。
使用ajax提交可以使用jQuery ajax - serialize() 方法使提交操作更简洁。

serialize() 方法通过序列化表单值，创建 URL 编码文本字符串。
您可以选择一个或多个表单元素（比如 input 及/或 文本框），或者 form 元素本身。
序列化的值可在生成 AJAX 请求时用于 URL 查询字符串中。

语法

$(selector).serialize()

下面是使用ajax方式提交主要代码：

$.ajax({
        url:'', //请求的地址
        data:$('#search_form').serialize(),
        beforeSend: function(){
            //ajax提交前执行的函数
            },
        success:function(result){//ajax提交成功返回数据后执行的函数
            $.each(result.orderList,function(i, v) {
                //遍历返回的json数据
            })
        }
        })

后端处理

Controller层对应请求方法主要代码：

String[] numbers = request.getParameterValues("Number");// 接收数组变量（名字相同，但值有多个的数据）
Map<String, String[]> map = new HashMap<String, String[]>();
map.put("numbers", numbers);//传入Mybatis参数为Map类型
List<Order> orderlist = 
orderService.queryOrderListByNumbers(map);//执行数据库操作，orderlist为查询结果

dao层，service层代码省略。

model层OrderMapping.xml主要代码：

<select id="queryOrderListByNumbers" resultMap="OrderMap" parameterType="java.util.Map">
    SELECT * FROM T_ORDER 
    WHRER ORDER_NUMBER IN
    <foreach collection="numbers" item="item" index="index"
        open="(" close=")" separator=",">
        #{item}
    </foreach>//mybatis遍历map参数
</select>
